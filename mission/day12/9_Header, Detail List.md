# 포켓몬 도감 - Header, Detail, List 컴포넌트 요약

## 1. Header 컴포넌트

- **역할:**  
  상단 타이틀(로고)과 검색창을 표시하며, 홈 이동 및 검색 이벤트를 처리합니다.
- **주요 기능:**
  - 메인 페이지에서만 검색창 노출
  - 타이틀 클릭 시 홈으로 이동
  - 검색창에서 엔터 또는 버튼 클릭 시 검색 실행
- **핵심 메서드:**
  - `template()`: 현재 페이지 상태에 따라 헤더 HTML 반환
  - `render()`: DOM 렌더링 및 이벤트 바인딩
  - `setState(newState)`: 상태 변경 및 재렌더링

---

## 2. PokemonDetail 컴포넌트

- **역할:**  
  선택한 포켓몬의 상세 정보를 보여줍니다.
- **주요 기능:**
  - 포켓몬의 이름, 이미지, 타입, 설명, 키, 분류, 몸무게 등 표시
- **핵심 메서드:**
  - `template()`: 상세 정보 화면의 HTML 반환
  - `render()`: DOM에 상세 정보 렌더링
  - `setState(newState)`: 상태 변경 및 재렌더링

---

## 3. PokemonList 컴포넌트

- **역할:**  
  포켓몬 목록을 렌더링하고, 각 포켓몬 및 타입 클릭 이벤트를 처리합니다.
- **주요 기능:**
  - 포켓몬 리스트 출력
  - 포켓몬 클릭 시 상세 페이지 이동
  - 타입 태그 클릭 시 해당 타입만 필터링
- **핵심 메서드:**
  - `template()`: 포켓몬 리스트 HTML 반환
  - `render()`: DOM에 리스트 렌더링 및 이벤트 바인딩
  - `setState(newState)`: 상태 변경 및 재렌더링

---

## 요약

- **Header**: 상단 UI, 검색 및 홈 이동 담당
- **PokemonDetail**: 포켓몬 상세 정보 표시
- **PokemonList**: 포켓몬 목록 및 필터링/상세 이동 담당

각 컴포넌트는 모두 `setState`를 통해 상태 변경 시 자동으로 화면을 갱신하며, 이벤트 바인딩을 통해 사용자 상호작용을 처리합니다.
